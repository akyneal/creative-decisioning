# Agent917 — Creative Decisioning for a Two-Sided Marketplace

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

A contextual bandit approach to lifecycle email optimisation, demonstrating how a
yoga-coaching marketplace can move from a single A/B champion to a per-segment policy
that maximises conversion — not clicks — with supply-side fairness guardrails.

> **DISCLAIMER — ILLUSTRATIVE SIMULATION**
>
> The company, marketplace, and all numbers in this repository are an **illustrative
> simulation** designed to demonstrate the methodology. This is **not real client data**.
> The data-generating process is intentionally constructed so that click-maximising and
> conversion-maximising creatives diverge; that divergence is the teaching point, not a
> found empirical result.

## The Methodology

### Four reframes

1. **Reward**: optimise *conversion* (booked-and-attended first paid class), not click-through
2. **Policy**: per-segment creative selection, not one champion for everyone
3. **Learning**: contextual bandit (LinTS with hierarchical priors), not sequential A/B tests
4. **Guardrails**: fairness constraints (minimum-exposure floors per coach tier), not
   unconstrained greedy optimisation

### The fairness layer

An unconstrained conversion-greedy policy concentrates ~71% of impressions on celebrity
coaches. The fairness layer redistributes traffic to keep all supply tiers viable at a cost
of ~0.1 percentage points of blended conversion.

## Not Included / Real-World Extensions

The following are explicitly **out of scope** to keep the repo LLM-free and dependency-light:

- **LLM-assisted variant generation** — using language models to generate headline/offer
  copy variants. In production, this would expand the arm space significantly.
- **Real-time feature store** — serving context features at email-render time with <50ms latency.
- **Online model updates** — streaming posterior updates instead of batch retraining.
- **Multi-objective optimisation** — jointly optimising conversion, LTV, and supply-side health.
- **Drift detection** — automated monitoring for distribution shift in context or reward.
